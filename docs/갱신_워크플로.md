# 일일 갱신 워크플로 (방식 B — Claude Code가 수행, 크롤 우선)

> 사용자가 **"오늘 뉴스 갱신해줘"** 하면 아래 순서대로 실행한다.
> 핵심 원칙: **잘 긁히는 곳은 크롤러로 긁고(토큰 0), Claude는 요약만** 한다.
> 안 긁히거나 자잘한 소스만 웹검색을 쓴다.

## ⭐ 매 갱신 필수 체크리스트 (사용자 요구 — 빠뜨리지 말 것)

1. **커뮤니티 반응은 상세하게 + 게시글 링크 필수.** 게임마다 긍정·부정(가능하면 중립까지)
   대표 의견을 **각각 2건 이상**, `topic`(쟁점) + `summary`(2~3문장 구체적으로) + `source`(아카/디시/공식 등)
   + **`source_url`(그 의견을 실제로 보여주는 게시글/스레드 URL)** 까지 채운다.
   `source_url`이 비면 프론트에 "게시글 보기" 링크가 안 뜬다 → **반드시 실제 URL을 넣는다.**
2. **이벤트·일정은 게임마다 상세 조사.** 진행 중 + 예정 이벤트를 모두. (자세히는 아래 §3b)
   특히 명조·스타레일만 풍부하고 **실버팰리스·브라운더스트2·니케·블아·명방·듀엣 등은 부실하니 보강.**
3. 참고자료에 **유튜브**(공식 채널·정보 채널의 업데이트 소개/생방송 요약)를 적극 활용한다.

## 사전 조건
- 백엔드 실행 중 (`http://localhost:8000`), `.env`의 `INGEST_TOKEN` 확인

## 1. 기계적 수집 (크롤러 — AI 토큰 0)
```
cd backend
PYTHONUTF8=1 .venv/Scripts/python.exe -m scripts.collect   # 시드 + Steam 공식 API 가격
PYTHONUTF8=1 .venv/Scripts/python.exe -m scripts.crawl      # 인벤 게임별 뉴스 → data/digest_<날짜>.json
```
- **Steam**: 공식 API로 가격/할인 (`steam_prices`).
- **인벤 크롤**: 게임별 뉴스 리스트를 긁어 제목·URL·날짜·**og:image**·**본문 스니펫**을 digest로 저장.

### 크롤 지원 게임
`nikke` 만 게임 전용 뉴스가 긁힌다(인벤 전용 포털 보유).
→ 인벤 `?site=<코드>`는 *전용 포털이 있는 게임*에만 게임별 뉴스를 주고, 나머지는
   인벤 전체 일반뉴스(LCK·영화 등 게임 무관)를 노출함. 그래서 `SITE_CODES`는 nikke만.

## 2. 웹검색 (보조 — 크롤 안 되는 나머지 전부)
nikke를 제외한 게임(`arknights·starrail·wuwa·brown2·bluearchive·silverpelis·duet` 등)은
**웹검색으로 수집**한다. 공식 SPA(쿠로게임즈·호요랩)·X 전용 속보도 웹검색.
참고자료: 공식 공지·인벤·나무위키 + **유튜브**(공식 채널 업데이트 소개/생방송 요약) 적극 활용.

> 기사 이미지: 인벤 등 뉴스 기사의 og:image는 기사별로 다르지만, 나무위키·공식 SPA·X로만
> 나온 소식은 기사별 이미지가 없어 게임 대표 키비주얼로 폴백된다(같은 이미지 반복 가능).

## 3. 커뮤니티 반응 (필수 · 상세 · 게시글 링크 포함)
```
PYTHONUTF8=1 .venv/Scripts/python.exe -m app.collectors.community <slug>
```
아카라이브/디시 인기글 제목으로 긍/중/부 분위기 정성 추정 + **실제 게시글 URL 확보**.

각 게임 `discussions[]`에 최소 **긍정 2 + 부정 2**(가능하면 중립 포함)를 넣되, 항목마다:
- `topic`: 쟁점 한 줄 (예: "신규 캐릭 성능", "보상 감소 논란")
- `sentiment`: 긍정 / 중립 / 부정
- `summary`: **2~3문장**으로 구체적 의견 정리 (왜 그렇게 반응하는지)
- `source`: 출처 라벨 (아카라이브 / 디시 / 공식 / 인벤 등)
- `source_url`: **그 의견을 보여주는 실제 게시글·스레드 URL** ← 절대 비워두지 말 것

> 비율(pos/neu/neg %)은 측정값이 아닌 **편집 추정치**. 의견·URL은 **실제 확인된 것만**.

## 3b. 이벤트·일정 (필수 · 상세 조사)
게임마다 **진행 중 + 예정** 이벤트/일정을 `events[]`에 충실히 채운다. 한두 개로 끝내지 말 것.
- 유형(`type`): 점검 / 업데이트 / 방송 / 콜라보 / 이벤트 / 시즌종료 등
- `start_date`(필수, ISO) · `end_date`(기간제면) · `title` · `source_url`
- 조사 소스: 공식 공지/SPA, 인벤·나무위키, **유튜브(공식 채널·정보 채널의 업데이트 소개/생방송 요약)**,
  X(웹검색). 신규 버전·픽업·복각·시즌 종료일 등 날짜가 잡힌 건 전부.
- **우선 보강 대상**(현재 부실): 실버팰리스 · 브라운더스트2 · 니케 · 블루아카이브 · 명일방주 · 듀엣.
  (명조·스타레일은 이미 양호)

## 4. AI 요약·편집 (Claude — 입력은 digest, 토큰 최소)
- `data/digest_<날짜>.json`(크롤 결과)을 읽고 핵심 뉴스 선별 + **본문 요약 작성**.
  - 각 기사: 제목·태그·중요도(imp)·본문요약(summary)·키비주얼(image=크롤 og:image).
- 크롤 미지원 게임은 웹검색 결과로 동일하게 작성.
- 게임별 한 줄 요약 + 커뮤니티 긍/중/부(추정) + 1면 리드 + 크로스게임 사건 알림.
- 게임 대표 키비주얼(`games[].image`)은 크롤 og:image 중 대표 1장 사용(기사 이미지 폴백).

> 정직성: 크롤/검색으로 **확인된 사실만** 기사화. 커뮤니티 비율은 편집 추정치.

## 5. 적재
```
curl -X POST http://localhost:8000/api/ingest \
  -H "Content-Type: application/json" -H "X-Ingest-Token: logia-dev-token" \
  -d @docs/sample_issue_<날짜>.json
```
같은 `issue_date` 재전송 시 그 날 편집판을 통째로 교체(idempotent).

## 6. 확인
`http://localhost:3000` 에서 메인/게임별/뉴스 모달/스팀할인/캘린더/아카이브 확인.
