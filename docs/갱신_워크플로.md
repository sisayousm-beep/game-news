# 일일 갱신 워크플로 (방식 B — Claude Code가 수행)

> 사용자가 Claude Code에서 **"오늘 뉴스 갱신해줘"** 라고 하면 아래 순서대로 실행한다.
> (별도 API 키·자동 스케줄 없음. Claude Code 세션 안에서 직접 수집·요약·적재한다.)

## 사전 조건
- 백엔드 실행 중: `cd backend && PYTHONUTF8=1 .venv/Scripts/python.exe -m uvicorn app.main:app --port 8000`
- `.env`의 `INGEST_TOKEN` 값 확인 (기본 `logia-dev-token`)

## 순서

### 1. 기계적 수집 (자동)
```
cd backend && PYTHONUTF8=1 .venv/Scripts/python.exe -m scripts.collect
```
- 시드(게임/스팀 목록) + **Steam 공식 API**로 전 추적게임 가격/할인 스냅샷 적재.
- 결과는 `steam_prices` 테이블에 누적 → 가격 추이 자동 기록.

### 2. 뉴스 수집 (Claude 도구)
게임별로 WebSearch / WebFetch 로 최신 공식 소식을 검색한다. 예) 공식 홈페이지·공지,
패치노트, 방송 예고, 로드맵, 콜라보, 이벤트. (X는 본문 수집 불가 → 검색/공식포럼으로 대체)

### 3. 커뮤니티 반응 (선택)
```
cd backend && PYTHONUTF8=1 .venv/Scripts/python.exe -m app.collectors.community <slug>
```
- 아카라이브/디시 인기글 제목을 받아 긍/중/부 분위기를 가늠한다(정성 추정).

### 4. AI 요약·편집 (Claude)
수집물을 바탕으로:
- 중복 제거(같은 공지가 여러 소스에 반복되면 하나로 병합)
- 게임별 핵심 뉴스(최대 10) / 주요 사건(최대 5) / 예정 이벤트(날짜순)
- 게임별 한 줄 요약 + 커뮤니티 긍/중/부 비율(추정)
- 1면 리드 스토리 + 편집국 리드(lede) + 크로스게임 사건 알림

> 정직성 규칙: 검색으로 **확인된 사실만** 기사화한다. 불확실하면 보수적으로 쓰거나 생략.
> 커뮤니티 비율은 측정값이 아닌 **편집 추정치**임을 전제로 한다.

### 5. 적재 (DB 기록)
요약 결과를 `IngestPayload` 형식 JSON으로 만들어 POST:
```
curl -X POST http://localhost:8000/api/ingest \
  -H "Content-Type: application/json" \
  -H "X-Ingest-Token: logia-dev-token" \
  -d @docs/sample_issue_2026-05-31.json
```
- 같은 `issue_date`로 다시 보내면 그 날 편집판을 통째로 교체(idempotent).
- 페이로드 예시: `docs/sample_issue_2026-05-31.json`

### 6. 확인
- 프론트(`http://localhost:3000`)에서 메인/게임별/스팀할인/캘린더/아카이브 확인.
